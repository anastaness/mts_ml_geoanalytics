# MTC-ItCup-2026

## 1. Описание кейса

Данный репозиторий содержит решение кейса **MTC IT Cup 2026**, связанного с обработкой, очисткой, сопоставлением и анализом геопространственных данных о зданиях из двух независимых источников.

Цель проекта — построить воспроизводимый пайплайн, который:

- выполняет первичный исследовательский анализ данных
- очищает и валидирует геометрию объектов
- объединяет данные из двух источников
- восстанавливает недостающие значения высоты зданий с помощью моделей машинного обучения
- проводит итоговую валидацию результатов
- формирует визуальные результаты: интерактивную карту и аналитические дашборды.

В рамках проекта были рассмотрены несколько моделей регрессии, включая:
- линейную регрессию
- случайный лес
- градиентный бустинг из `sklearn`
- `CatBoost`

Итоговые результаты дополнительно визуализируются в виде:
- EDA-дашборда
- validation-дашборда
- интерактивной карты
- презентации в формате .pdf

## 3. Шаги решения

Работа над кейсом была организована как последовательный пайплайн из нескольких этапов.

### Шаг 1. Первичный анализ данных
На первом этапе был проведен EDA двух исходных источников. Анализировались:
- структура таблиц
- состав признаков
- пропуски
- базовые статистики
- корректность геометрий
- наличие подозрительных дубликатов
- качество числовых признаков

Результат этапа:
- понимание состава и качества исходных данных
- формирование гипотез о проблемных местах
- подготовка базы для очистки

### Шаг 2. Очистка данных
На этом этапе выполнялась очистка источников:
- исправление и фильтрация проблемных геометрий
- удаление или маркировка некорректных объектов
- унификация структуры данных
- сохранение промежуточных очищенных наборов

Результат этапа:
- формирование очищенных геоданных для последующего сопоставления

### Шаг 3. Слияние данных
После очистки выполнялось сопоставление объектов между двумя источниками и построение итогового объединенного датасета.

На этом этапе:
- анализировались пространственные совпадения
- строились признаки для объединения
- формировались компоненты соответствия
- вычислялись агрегаты по объектам и соседям
- подготавливался единый датасет для моделирования

Результат этапа:
- единый набор объектов с признаками из обоих источников

### Шаг 4. Сравнение моделей
Для восстановления недостающих высот зданий были протестированы несколько моделей:
- линейная регрессия
- случайный лес
- градиентный бустинг
- CatBoost

Для каждой модели анализировались:
- качество предсказаний
- устойчивость результатов
- интерпретируемость
- пригодность к задаче заполнения пропусков

Результат этапа:
- выбор наиболее подходящей модели для финального восстановления высот

### Шаг 5. Финальное обучение
После выбора модели было выполнено финальное обучение на подготовленном датасете и построение итоговых предсказаний.

Результат этапа:
- заполненная колонка высоты
- признаки происхождения высоты
- маркировка предсказанных значений

### Шаг 6. Валидация результатов
После моделирования был выполнен отдельный этап пост-ML валидации итогового датасета.

Проверялись:
- согласованность геометрии
- полнота высот
- корректность флагов заполнения
- отсутствие перезаписи наблюдаемых значений
- пространственная правдоподобность результатов
- экстремальные выбросы

Результат этапа:
- финальный validation dashboard
- интерпретация надежности итогового набора

### Шаг 7. Визуализация
Финальные результаты были представлены в нескольких формах:
- интерактивная карта
- EDA dashboard
- validation dashboard
- поясняющие markdown-материалы

## 5. Назначение основных директорий

```data/raw/```
Содержит исходные данные. По условиям задачи сами датасеты не публикуются, поэтому в репозитории хранится только структура папок.

```data/interim/```
Содержит промежуточные результаты: очищенные слои, промежуточные таблицы, результаты объединения.

```data/processed/```
Содержит итоговые данные после подготовки и моделирования, используемые для валидации и визуализации.

```data/map/```
Содержит интерактивные карты.

```notebooks/```
Содержит исследовательские ноутбуки, отражающие логику решения по этапам.

```src/```
Содержит основной воспроизводимый код для очистки, объединения, обучения и рендера отчетов.

```docs/```
Содержит итоговые пояснительные материалы и готовые PDF.

## 6. Запуск проекта

Ниже приведена подробная инструкция по запуску проекта на локальной машине.

### 6.1. Установить Python

Если Python еще не установлен:

- Перейдите на официальный сайт Python.
- Скачайте версию Python 3.10+ или 3.11.
- Установите Python в систему.
- Во время установки на Windows обязательно включите опцию:
```Add Python to PATH```

После установки проверьте, что Python доступен в терминале.

Для Windows:
```python --version```

Для Linux / macOS:
```python3 --version```


### 6.2. Клонировать репозиторий

```
git clone https://github.com/hep2014/MTC-ItCup-2026.git
cd MTC-ItCup-2026
```

### 6.3. Создать виртуальное окружение

Для Windows:
```
python -m venv .venv
.venv\Scripts\activate
```

Для Linux / macOS: 
```
python3 -m venv .venv
source .venv/bin/activate
```

### 6.4. Установить зависимости
```
pip install -r requirements.txt
```

### 6.5. Установить браузер Chromium для Playwright

Для сборки PDF-дашбордов из HTML требуется установленный Chromium:
```
python -m playwright install chromium
```
Если используется Linux/macOS:
```
python3 -m playwright install chromium
```

### 6.6. Подготовить данные

По условиям кейса исходные датасеты не публикуются в репозитории.

Перед запуском необходимо вручную поместить нужные файлы в соответствующую директорию:
```
data/raw/
```

### 6.7. Запуск ноутбуков

Для работы с ноутбуками можно использовать Jupyter Notebook или JupyterLab.
Установка Jupyter при необходимости:
```
pip install notebook
```
или
```
pip install jupyterlab
```
Запуск:
```
jupyter notebook
```
или
```
jupyter lab
```
После этого можно открывать ноутбуки из папки notebooks/ и выполнять их по порядку.

Рекомендуемый порядок прохождения:

- 01_EDA.ipynb
- 02_Cleaning_data.ipynb
- 03_Data_merge.ipynb
- 04_linear_regression.ipynb
- 05_random_forest.ipynb
- 06_gradient_boosting.ipynb
- 07_catboost.ipynb
- 08_final_training.ipynb
- 09_merge.ipynb
- 10_validation.ipynb
- 11_interactive_map.ipynb

### 6.8. CLI-запуск Python-скриптов

Помимо ноутбуков, часть проекта поддерживает запуск через CLI.

#### 6.8.0 Создание файла для построения дэшборда

Скрипт: ```src/parse_eda.py```

Общий шаблон запуска:
```
python parse_eda.py
```

#### 6.8.1. Рендер EDA dashboard

Скрипт: ```src/reporting/report_render.py```

Назначение:

- читает Excel-файл с результатами EDA
- строит HTML-дашборд
- экспортирует итоговый PDF через Playwright

Общий шаблон запуска:
```
python src/reporting/report_render.py --xlsx <путь_к_xlsx> --template-dir <путь_к_папке_шаблонов> --template-name eda_dashboard.html.j2 --project-title "EDA Quality Dashboard" --logo <путь_к_logo.png> --out-html <путь_к_html> --out-pdf <путь_к_pdf>
```
Пример запуска из корня проекта:
```
python src/reporting/report_render.py ^
  --xlsx data/processed/eda_outputs_refactored.xlsx ^
  --template-dir src/reporting/templates ^
  --template-name eda_dashboard.html.j2 ^
  --project-title "EDA Quality Dashboard" ^
  --logo src/reporting/templates/logo.png ^
  --out-html docs/eda_dashboard.html ^
  --out-pdf docs/03_eda_dashboard.pdf
```

#### 6.8.2. Рендер validation dashboard

Скрипт: ```src/reporting/render_validation_dashboard.py```

Назначение:

- читает итоговый parquet-файл после моделирования
- строит post-ML validation dashboard
- экспортирует HTML и PDF

Общий шаблон запуска:
```
python src/reporting/render_validation_dashboard.py --parquet <путь_к_parquet> --template-dir <путь_к_шаблонам> --template-name validation_dashboard.html.j2 --project-title "Validation Quality Dashboard" --logo <путь_к_logo.png> --out-html <путь_к_html> --out-pdf <путь_к_pdf>
```

Пример запуска из корня проекта:
```
python src/reporting/render_validation_dashboard.py ^
  --parquet data/processed/final_validation_dataset.parquet ^
  --template-dir src/reporting/templates ^
  --template-name validation_dashboard.html.j2 ^
  --project-title "Validation Quality Dashboard" ^
  --logo src/reporting/templates/logo.png ^
  --out-html docs/validation_dashboard.html ^
  --out-pdf docs/04_validation_results.pdf
```
